# Инструкция по деплою на Timeweb

## Зачем нужен сервер?

Для статического сайта нужен веб-сервер (Apache/Nginx), который:
- Раздает HTML, CSS, JS файлы браузерам
- Обрабатывает запросы и маршрутизацию (через `.htaccess` для SPA)
- Обеспечивает безопасность и производительность

Timeweb предоставляет Apache/Nginx автоматически при размещении сайта.

## Размещение нескольких сайтов на одном сервере

**Да, можно использовать один сервер для нескольких доменов!** 

На вашем сервере уже размещен сайт `elenaquestionnaire.ru`. Для второго домена `elenafitmagic.ru` нужно:

1. **В панели Timeweb привязать домен к серверу:**
   - Домены → `elenafitmagic.ru` → Настройки → Привязать к серверу "Questionnaire"
   - Timeweb автоматически создаст директорию для второго домена

2. **Определить директорию для второго домена:**
   - Обычно это `public_html/elenafitmagic.ru/` или отдельная директория
   - Проверьте в панели Timeweb в разделе "Домены" → `elenafitmagic.ru` → "Директория сайта"

3. **Загрузить файлы в эту директорию** (не в `public_html/`, чтобы не перезаписать первый сайт!)

## Подготовка проекта

Проект уже настроен для деплоя на корневой домен (`base: "/"` в `vite.config.ts`).

## Деплой на Timeweb

### Вариант 1: Автоматический деплой через скрипт (рекомендуется)

**Через Node.js скрипт:**
```bash
npm install  # Установит ssh2-sftp-client если еще не установлен
npm run deploy
```

Скрипт автоматически:
1. Соберет проект (`npm run build`)
2. Запросит данные для SFTP подключения
3. Загрузит все файлы из `dist/` на сервер

**Через Bash скрипт:**
```bash
npm run deploy:bash
# или напрямую:
./scripts/deploy.sh
```

Требует установки `lftp`: `brew install lftp` (macOS)

### Вариант 2: Ручной деплой через FTP/SFTP

1. Соберите проект:
```bash
npm run build
```

2. Войдите в панель управления Timeweb
3. Откройте раздел "Домены" → `elenafitmagic.ru` → найдите "Директория сайта" (например, `public_html/elenafitmagic.ru/`)
4. Откройте раздел "Файловый менеджер" или используйте FTP-клиент (FileZilla, WinSCP)
5. Подключитесь к серверу по FTP/SFTP (данные в панели Timeweb)
6. **Важно:** Загрузите все содержимое папки `dist/` в директорию второго домена (НЕ в `public_html/`, чтобы не затронуть первый сайт!)

**Важно:** 
- Убедитесь, что файл `.htaccess` загружен в директорию второго домена
- Не загружайте файлы в `public_html/` напрямую - это перезапишет первый сайт!

## Настройка SSL сертификата через Let's Encrypt

Поскольку в панели Timeweb требуется платный балансировщик для бесплатного SSL, настроим через certbot напрямую на сервере.

### Шаг 1: Подключение к серверу по SSH

1. В панели Timeweb найдите данные для SSH подключения
2. Подключитесь к серверу:
```bash
ssh ваш_пользователь@ваш_сервер
```

### Шаг 2: Установка SSL через скрипт

1. Загрузите скрипт на сервер:
```bash
# С вашего компьютера
scp scripts/setup-ssl.sh ваш_пользователь@ваш_сервер:/tmp/
```

2. На сервере выполните:
```bash
chmod +x /tmp/setup-ssl.sh
sudo /tmp/setup-ssl.sh
```

Скрипт автоматически:
- Установит certbot (если не установлен)
- Получит SSL сертификат от Let's Encrypt
- Настроит Apache для работы с HTTPS
- Настроит автообновление сертификата

### Альтернатива: Ручная установка через certbot

Если скрипт не работает, выполните вручную на сервере:

```bash
# Установка certbot (для Debian/Ubuntu)
sudo apt-get update
sudo apt-get install -y certbot python3-certbot-apache

# Получение сертификата
sudo certbot --apache -d elenafitmagic.ru -d www.elenafitmagic.ru

# Настройка автообновления
sudo crontab -e
# Добавьте строку:
0 0,12 * * * certbot renew --quiet
```

### Проверка SSL

После установки проверьте:
```bash
# Проверка сертификата
sudo certbot certificates

# Тест автообновления
sudo certbot renew --dry-run
```

## Настройка для SPA (Single Page Application)

Файл `.htaccess` уже создан в папке `public/` и будет скопирован в `dist/` при сборке. Он обеспечивает правильную работу роутинга React Router.

## Проверка после деплоя

1. Откройте `https://elenafitmagic.ru` - должна открыться главная страница
2. Проверьте переходы по внутренним ссылкам (например, `/privacy`)
3. Убедитесь, что все ресурсы (CSS, JS, изображения) загружаются корректно
4. Проверьте работу в режиме инкогнито

## Возможные проблемы

### Проблема: 404 при переходе по прямым ссылкам
**Решение**: Убедитесь, что файл `.htaccess` загружен в корневую директорию сайта

### Проблема: Смешанный контент (Mixed Content)
**Решение**: Убедитесь, что все внешние ресурсы используют HTTPS

### Проблема: Старые файлы кешируются
**Решение**: Очистите кеш браузера или используйте режим инкогнито
