# Проблема с делегированием DNS для elenafitmagic.ru

## Проблема

DNS возвращает `NXDOMAIN` - домен не найден. Это означает, что домен не делегирован на NS-серверы Timeweb.

## Решение

### 1. Проверьте NS-серверы домена

В панели Timeweb: Домены → `elenafitmagic.ru` → Управление → NS-серверы

Должны быть указаны NS-серверы Timeweb:
- `ns1.timeweb.ru`
- `ns2.timeweb.ru`
- `ns3.timeweb.org`
- `ns4.timeweb.org`

### 2. Проверьте делегирование у регистратора

Если домен зарегистрирован не в Timeweb, нужно настроить делегирование у регистратора:

1. Войдите в панель регистратора домена
2. Найдите настройки DNS/NS-серверов для домена
3. Укажите NS-серверы Timeweb:
   - `ns1.timeweb.ru`
   - `ns2.timeweb.ru`
   - `ns3.timeweb.org`
   - `ns4.timeweb.org`

### 3. Проверка делегирования

После настройки проверьте:

```bash
# Проверка NS-записей
dig elenafitmagic.ru NS

# Должны вернуться NS-серверы Timeweb
# Если возвращаются другие NS - делегирование еще не применилось
```

### 4. Время распространения

Делегирование DNS может занять:
- Минимум: 15-30 минут
- Обычно: 2-6 часов
- Максимум: до 48 часов

### 5. Альтернатива: Использование DNS регистратора

Если не хотите менять NS-серверы, можно:
1. Оставить NS-серверы регистратора
2. В панели регистратора добавить A-запись:
   - Тип: A
   - Имя: @ (или elenafitmagic.ru)
   - Значение: <SERVER_IP>
   - TTL: 600

3. И для www:
   - Тип: A
   - Имя: www
   - Значение: <SERVER_IP>
   - TTL: 600

## Проверка текущего состояния

Выполните на вашем компьютере:

```bash
# Проверка NS-серверов
dig elenafitmagic.ru NS

# Проверка кто отвечает за домен
whois elenafitmagic.ru | grep -i "name server\|nserver"
```

Если NS-серверы не Timeweb - нужно настроить делегирование у регистратора.
